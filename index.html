物件導向(六) Get & Set 存取子

Get & Set 存取子的基本使用

Get & Set 存取子是用來設定 Class 的 Property，根據該屬性存取設定會有以下三種設定方式。

  1. 唯讀: 用 Get 取值，將要回傳的值 return 回去

  2. 唯寫: 用 Set 寫入他的值，寫入的值會存在 value

  3. 可以讀跟寫: 同時寫上 Get & Set

Get & Set 存取子的簡化版本

看到這裡您可能會想也太麻煩了吧!定義一個屬性除了 public 屬性之外，還需要多定義一組對應的 private 屬性。所以在 C# 3.0 增加了自動實作屬性（automatically implemented properties），讓我們可以用更簡單的語法，就能設定出能讀能寫的 property。

若是要設定屬性唯讀，在 set 存取子前加上 private 就可以。

Get & Set 存取子變化範例

若是我們要在 Car Class 中建立一個汽油存量的屬性 Gasoline，其屬性值不能低於 0。我們可以再 Gasoline 屬性的 set 存取子進行判斷，如果欲存入的 value < 0，則會直接存入 0，避免油量為負數的狀況。

測試1: 當 Gasoline 存入的值為 100 時。

When set Gasoline 100: 100

測試2: 當 Gasoline 存入的值為 -42 時。

When set Gasoline -42: 0

透過 get set 存取器，就能輕鬆在屬性端設定存取限制。防呆避免不是預期的值被存入屬性中。

物件導向(五) 封裝性 Public Private Protected

封裝性 (Encapsulation) 是物件導向三大特性之一。若是要理解封裝性，可以想像當我們在駕駛汽車時想加速的話，我們不需要理解引擎如何運作將動力傳輸到輪胎上，只需要踩下油門踏板就好，因為加速的知識已經被 封裝 起來了。再舉一個例子我們想拍照的時候不需要去理解鏡頭是如何捕捉影像在儲存至手機裡，只需要按快門就好，因為按快門至儲存影像這中間的運作，也已經被 封裝 起來了，透過這兩個例子應該會對封裝性有初步的理解吧!

存取修飾詞

而要善用物件導向的封裝特性，我們首先需要認識一些 存取修飾詞，下面將簡單介紹幾個常見的存取修飾詞。

  • public: 被設定的 class 或其成員可以被 任何人 所存取。
  • private: 被設定的 class 或成員，只能在 自己的類別結構中 被存取。
  • protected: 被設定的 class 或成員，只可由 相同類別或繼承自該類別 的程式碼進行存取。

若想知道更多的存取修飾字資訊，可以參考 MSDN 存取修飾詞 。

簡介範例的 code example

我們試著把第一段簡介中油門踏板範例透過程式實例化出來，首先會有一個類別 Car，記錄著車輛的各種功能，其中 public 的功能是可以直接被外部所呼叫的，而 private 的功能則只能被 Car 類別內的其他功能所呼叫。

再建立一個駕駛者的類別 Driver，其中有一個方法是踩下油門，而踩下油門後會做的動作就是去呼叫 Car 類別的 public method 加速()。

如此一來 Driver 想加速時只要踩下油門就會加速，而 Car 類別中其他 private method Driver 都不需要了解(也不能呼叫)，因為全部都被 封裝 在加速() method 了。

物件導向(四) static修飾字 靜態類別和靜態類別成員

無論是程式新手或是有一些開發經驗的工程師，一定都會在範例程式或是專案中常常看到 static 關鍵字。本篇文章將簡單介紹 static 常見的使用情境，還有在使用時需要注意的事項。

靜態方法

靜態方法常出現在專案中的共用方法，像是 helper class 或是 type extention，最方便的地方在於靜態方法不需要被實體化，就可以被使用。下面的範例將示範傳統非靜態類方法與靜態方法被呼叫的方式:

建立一個計算類別，其中包含 Plus (靜態方法) 以及 Minus (非靜態方法)

呼叫傳統的非靜態方法需先將 Calculate 類別實體化成物件，再由物件呼叫方法。

output: 10

呼叫靜態方法，不須要將類別實體化為物件，就可以直接透過類別呼叫。

output: 30

靜態變數

除了方法外，變數也可以在宣告時加入 static 修飾字，一樣是不需要實體化類別就可以被使用，但要特別注意的是，所有該類別實體化的物件，都可以改變靜態變數，而且這個靜態變數將被所有物件共用，下面將示範一個靜態變數的使用情境。

家同共用帳戶範例:

建立一個 Family class，其中有靜態變數 accountMoney 記錄家庭帳戶的金額。還有兩個非靜態方法 saveMoney()、spendMoney() 來異動 accountMoney 的數字。

在 Dad 做了存錢，Mom 做了花錢的動作後，因為 accountMoney 為靜態變數，所以會被所有物件 (Dad、Mon) 共用。也因 accountMoney 為靜態變數，所以在輸出 accountMoney 時，我們要直接透過類別 (Family) 來呼叫。

輸出結果: \
爸爸存了 100 元 \
現在帳戶金額是 : 100 \
媽媽花了 50 元 \
現在帳戶金額是 : 50

靜態類別

若我們可以確定該類別內的所有成員 (property、method…) 皆為靜態類型，也可以直接將 class 設計為靜態類別。好處在於可以確保類別不會被實體化，且類別內容若出現非靜態成員，編輯器也會在編譯前就先報錯提醒。要設計一個靜態類別，需要注意以下事項:

  • 只能包含靜態類別成員
  • 不允許被實體化為物件
  • 不能被其他類別所繼承

而靜態類別的寫法也相當簡單，在 class 前加上 static 關鍵字即可。

物件導向(二) 實質型別(value type) 與參考型別(reference type)

C# 依照記憶體存取方式的不同，將型別大致分為實質型別 (value type) 與參考型別 (reference type)。也因為這兩種型別在執行存取時運作的模式不一樣，觀念不夠清晰的情況下，常會發生執行結果與預期不相符的情況。這篇文章將簡單說明 value type 與 reference type 在執行記憶體存取的運作情形。

記憶體位置 stack & heap

在說明兩種型別存取過程前，需要先簡單了解記憶體有兩個重要區塊:

  • stack: 用於靜態記憶體配置，必須在編譯時期為已知的資料。
  • heap: 用於動態記憶體配置，使用這裡的記憶體主要是用在編譯時期還不知道大小或個數的資料。

這部分非本章節重點，只要先大致知道記憶體有這兩個不同區塊即可。

實質型別 Value Type

C# 中絕大部分的型別都屬於 value type，像是 int、bool、double、decimel…，在記憶體中存取的方式比較單純，下面將用一個簡單的例子來拆解 value type 在記憶體中存取的逐步運作。

  1. 當執行 int a = 0 時，記憶體狀況如下:

    在 stack 新增一個區塊 a，填入值為 0。

  2. 當執行 int b = a 時，記憶體狀況如下:

    在 stack 新增一個區塊 b，填入值為和 a 相同的 0。

  3. 當執行 b = 20 時，記憶體狀況如下:

    b 原始的值 0 被新賦予值 20 取代。

  4. 所以程式執行最終的輸出結果:

    a = 0、b = 20。

參考型別 Reference Type

參考型別在 C# 中最指標性的型別就是物件 (Object)，下面也會用一個簡單的例子來拆解 reference type 在記憶體中存取的逐步運作。

  1. 先建立一個 Car class，有兩個屬性 brand (string) 及 speed (int)。

  2. new 一個 Car，命名為 car1，並將其屬性賦予值，此時記憶體狀態如下:

    car1 在 stack 建立了一組 car1 address，指向到儲存於 heap 的物件內容。

  3. Car car2 = car1。 宣告一個 car2 並將 car1 的值賦予給 car2，此時記憶體狀態如下:

    car2 在 stack 新增了一個區塊，並將 car1 的值 car1 address 賦予給 car2，所以此時 car1、car2 在 stack 中的儲存內容都是 car1 address，指向 heap 中同一區塊。

  4. 修改 car2 的屬性 brand、speed，此時記憶體狀況如下:

    屬性變更會直接到 heap 中修改屬性值，所以 heap 區塊中的 brand 變為 Mazda，speed 變為 60。

  5. 由於 car1、car2 都指向相同的 heap 區塊，所以輸出結果皆為

    Mazda品牌的汽車正以60公里的速度奔馳

物件導向(一) 物件與類別

要學習物件導向的第一步，就是要先釐清 物件(Object) 與 類別(Class) 的關係。
以下整理了幾個網路上比喻物件類別之間關係的例子:

  1. 類別是建築物的設計圖，而物件則是依照設計圖設計出來的建築物。
  2. 類別是一本武林秘笈，而物件則是精通武林秘笈的武林高手。
  3. 物件是實體化後的類別。

而我自己的理解是類別是虛擬的，需要將類別實體化(new出來)變成物件，才能被使用，下面將使用一段簡單的程式碼來示範何謂類別，何謂物件。

程式碼說明:

  1. 首先建立一個 Car 類別。
  2. Car 類別中包含三個屬性( brand、color、speed )及一個方法( Run )。
  3. 在主程式 Main 中實體化一個 Car 類別的物件名為 toyota。
  4. 賦予 toyota 這個物件的屬性值。
  5. 執行 Run() 方法，並將結果輸出。

以上就是物件與類別的基本介紹，可以將重點歸納為下面幾個項目:

  • 物件是實體化後的類別。
  • 類別中包含屬性( property )與方法( method )。
  • 要產生物件必須要將類別 new 出來。

委派的宣告

簡單來說，委派就是將方法當成參數傳進另一個方法中。上面這個說法可能過於簡短有點抽象，我們可以先來了解一下委派宣告的方式。

委派的規則

將委派當作參數傳到其他方法

看到這裡您可能會想，那我就直接執行 MaMaCook() 就好啦，何須多此一舉宣告一個委派來執行。但呼應文章開頭所說的「委派就是將方法當成參數傳進另一個方法中」，在設計類別時，可能會碰到某個方法需要依照不同情境有不同的處理，但又不想把該處理用 if 或 switch 來處理，因為有可能變化很多，又或是無法預先得知處理的規則，這時我們就可以將方法指派給 howToCook，再將 howToCook 當作參數傳進其他 function。

物件導向(五) 封裝性 Public Private Protected

封裝性 (Encapsulation) 是物件導向三大特性之一。若是要理解封裝性，可以想像當我們在駕駛汽車時想加速的話，我們不需要理解引擎如何運作將動力傳輸到輪胎上，只需要踩下油門踏板就好，因為加速的知識已經被 封裝 起來了。再舉一個例子我們想拍照的時候不需要去理解鏡頭是如何捕捉影像在儲存至手機裡，只需要按快門就好，因為按快門至儲存影像這中間的運作，也已經被 封裝 起來了，透過這兩個例子應該會對封裝性有初步的理解吧!

存取修飾詞

而要善用物件導向的封裝特性，我們首先需要認識一些 存取修飾詞，下面將簡單介紹幾個常見的存取修飾詞。

  • public: 被設定的 class 或其成員可以被 任何人 所存取。
  • private: 被設定的 class 或成員，只能在 自己的類別結構中 被存取。
  • protected: 被設定的 class 或成員，只可由 相同類別或繼承自該類別 的程式碼進行存取。

若想知道更多的存取修飾字資訊，可以參考 MSDN 存取修飾詞 。

簡介範例的 code example

我們試著把第一段簡介中油門踏板範例透過程式實例化出來，首先會有一個類別 Car，記錄著車輛的各種功能，其中 public 的功能是可以直接被外部所呼叫的，而 private 的功能則只能被 Car 類別內的其他功能所呼叫。

再建立一個駕駛者的類別 Driver，其中有一個方法是踩下油門，而踩下油門後會做的動作就是去呼叫 Car 類別的 public method 加速()。

如此一來 Driver 想加速時只要踩下油門就會加速，而 Car 類別中其他 private method Driver 都不需要了解(也不能呼叫)，因為全部都被 封裝 在加速() method 了。

C# Delegate 委派的使用情境與簡介

委派的宣告

簡單來說，委派就是將方法當成參數傳進另一個方法中。上面這個說法可能過於簡短有點抽象，我們可以先來了解一下委派宣告的方式。

/// <summary>
/// 烹調方式 Delegate
/// </summary>
/// <param name="食材"></param>
/// <returns>煮好的佳餚</returns>

public delegate string HowToCook(string 食材);

委派的規則

而上面宣告的委派可以持有一個或多個方法，但前題是持有的方法必須與委派有相同的輸入和輸出，否則將無法編譯成功。下面將提供4個方法範例，媽媽煮、爸爸煮、妹妹煮及阿嬤煮。

// 媽媽煮
public static string MaMaCook(string 食材)
{
    Console.WriteLine($"先清理{食材}");
    Console.WriteLine("熱鍋後爆香");
    Console.WriteLine("快炒3分鐘");

    return $"一盤美味的{食材}料理";
}

// 爸爸煮
public static string PaPaCook(string 食材)
{
    Console.WriteLine($"先清理{食材}");
    Console.WriteLine("發生火災");

    return $"一盤焦黑的{食材}料理";
}

// 妹妹煮
public static void SisterCook(string 食材)
{
    Console.WriteLine($"我不會煮{食材}");
}

// 阿嬤煮
public static string GrandMaCook(string 食材)
{
    Console.WriteLine($"阿嬤覺得不夠鹹，加一些調味料在{食材}料理上");
    return $"一盤重口味的{食材}料理";
}

指派方法給委派

可以使用下面方式來將方法指派給 Delegate。

// 產生委派的參考
HowToCook howToCook;

// 將 媽媽煮 指派給 howToCook
howToCook = MaMaCook;

// 也可以將 爸爸煮 指派給 howToCook
howToCook = PaPaCook;

// 將 妹妹煮 指派給 howToCook 時會發生錯誤
// 因為 妹妹煮 回傳值是 void，不符合 HowToCook 回傳值須為 string 的條件
howToCook = SisterCook; // 會編譯失敗

委派的運作原理

將 媽媽煮 指派給委派的參考 howToCook 之後，我們直接執行 howToCook 將會得到和執行 媽媽煮 同樣的結果，而實際上，執行委派參考 howToCook 就是去執行 媽媽煮。

// 產生委派的參考
HowToCook howToCook;

// 將 媽媽煮 指派給 howToCook
howToCook = MaMaCook;

string MaMaCookResult = MaMaCook("鮮魚");
string HowToCookResult = howToCook("鮮魚");

Console.WriteLine($"MaMaCookResult : {MaMaCookResult}");
// Output : MaMaCookResult : 一盤美味的鮮魚料理

Console.WriteLine($"HowToCookResult : {HowToCookResult}");
// Output : HowToCookResult : 一盤美味的鮮魚料理

將委派當作參數傳到其他方法

看到這裡您可能會想，那我就直接執行 MaMaCook() 就好啦，何須多此一舉宣告一個委派來執行。但呼應文章開頭所說的「委派就是將方法當成參數傳進另一個方法中」，在設計類別時，可能會碰到某個方法需要依照不同情境有不同的處理，但又不想把該處理用 if 或 switch 來處理，因為有可能變化很多，又或是無法預先得知處理的規則，這時我們就可以將方法指派給 howToCook，再將 howToCook 當作參數傳進其他 function。

例如下方範例的 GetFood 方法，其中可能有數十種，甚至數百種的烹煮食物邏輯，我們若把所有邏輯寫在同個 function 中，這個 GetFood 方法勢必會變得又臭又長，不好維護。所以我們將烹煮食物邏輯抽出，包裝在委派 HowToCook 中，並當成參數傳進 GetFood 方法，如此一來， GetFood 方法就只要專注於自己的輸出功能上，不用去在意烹煮食物邏輯。

public static void GetFood(int 人數, HowToCook 烹調方式)
{
    var food = 烹調方式("鮮魚");
    Console.WriteLine($"{food}給{人數}個人吃");
}

我們只要將 媽媽煮 指派給 howToCook，並呼叫，GetFood(4, howToCook)就可以得到以下的結果。

多重委派

委派可以被指派一個或多個方法，上面的範例都是一個委派被指派一個方法，下面這個範例則是示範委派參考 howToCook 如何被指派多個方法。

static void Main(string[] args)
{
    // 產生委派的參考
    HowToCook howToCook;

    // 指派的一個方法 媽媽煮 
    howToCook = MaMaCook;

    // 指派第二個方法 阿嬤煮，透過 += 運算子
    howToCook += GrandMaCook;

    // 執行 GetFood 方法，並將委派參考 howToCook 當作參數傳入
    GetFood(4, howToCook);
    Console.Read();
}

在使用多重委派還有幾個特性需要注意:

  • 被指派的任何方法，一定都要符合 Delegate 宣告的 input output 規範。
  • 會依照被指派的順序依序執行。
  • 可以接受同樣的方法被重複指派。
  • 當執行委派參考時，得到的回傳值，會是最後一個被指派方法的回傳值。

完整範例程式碼

以下是本篇文章範例的完整程式碼，也會同步發行到 GitHub，有興趣的朋友可以 Clone 下來執行看看。

class Program
{
    /// <summary>
    /// 烹調方式 Delegate
    /// </summary>
    /// <param name="食材"></param>
    /// <returns>煮好的佳餚</returns>
    public delegate string HowToCook(string 食材);

    /// <summary>
    /// 媽媽煮
    /// </summary>
    /// <param name="食材"></param>
    /// <returns>煮好的佳餚</returns>
    public static string MaMaCook(string 食材)
    {
        Console.WriteLine($"先清理{食材}");
        Console.WriteLine("熱鍋後爆香");
        Console.WriteLine("快炒3分鐘");

        return $"一盤美味的{食材}料理";
    }

    /// <summary>
    /// 爸爸煮
    /// </summary>
    /// <param name="食材"></param>
    /// <returns>煮好的佳餚</returns>
    public static string PaPaCook(string 食材)
    {
        Console.WriteLine($"先清理{食材}");
        Console.WriteLine("發生火災");

        return $"一盤焦黑的{食材}料理";
    }

    /// <summary>
    /// 妹妹煮
    /// </summary>
    /// <param name="食材"></param>
    /// <returns>煮好的佳餚</returns>
    public static void SisterCook(string 食材)
    {
        Console.WriteLine($"我不會煮{食材}");
    }

    /// <summary>
    /// 阿嬤煮
    /// </summary>
    /// <param name="食材"></param>
    /// <returns>煮好的佳餚</returns>
    public static string GrandMaCook(string 食材)
    {
        Console.WriteLine($"阿嬤覺得不夠鹹，加一些調味料在{食材}料理上");
        return $"一盤重口味的{食材}料理";
    }

    /// <summary>
    /// 得到煮好的食物
    /// </summary>
    /// <param name="人數"></param>
    /// <param name="烹調方式"></param>
    public static void GetFood(int 人數, HowToCook 烹調方式)
    {
        var food = 烹調方式("鮮魚");
        Console.WriteLine($"{food}給{人數}個人吃");
    }

    private static void Main(string[] args)
    {
        //// 產生委派的參考
        //HowToCook howToCook;

        //// 將 媽媽煮 指派給 howToCook
        //howToCook = MaMaCook;

        //string MaMaCookResult = MaMaCook("鮮魚");
        //string HowToCookResult = howToCook("鮮魚");

        //Console.WriteLine($"MaMaCookResult : {MaMaCookResult}");
        //// Output : MaMaCookResult : 一盤美味的鮮魚料理

        //Console.WriteLine($"HowToCookResult : {HowToCookResult}");
        //// Output : HowToCookResult : 一盤美味的鮮魚料理

        HowToCook howToCook;
        howToCook = MaMaCook;
        howToCook += GrandMaCook;

        GetFood(4, howToCook);
        Console.Read();
    }
}

物件導向(二) 實質型別(value type) 與參考型別(reference type)

發表於 2020-01-12 更新於 2023-09-05

C# 依照記憶體存取方式的不同，將型別大致分為實質型別 (value type) 與參考型別 (reference type)。也因為這兩種型別在執行存取時運作的模式不一樣，觀念不夠清晰的情況下，常會發生執行結果與預期不相符的情況。這篇文章將簡單說明 value type 與 reference type 在執行記憶體存取的運作情形。

記憶體位置 stack & heap

在說明兩種型別存取過程前，需要先簡單了解記憶體有兩個重要區塊:

  • stack: 用於靜態記憶體配置，必須在編譯時期為已知的資料。
  • heap: 用於動態記憶體配置，使用這裡的記憶體主要是用在編譯時期還不知道大小或個數的資料。

這部分非本章節重點，只要先大致知道記憶體有這兩個不同區塊即可。

實質型別 Value Type

C# 中絕大部分的型別都屬於 value type，像是 int、bool、double、decimel…，在記憶體中存取的方式比較單純，下面將用一個簡單的例子來拆解 value type 在記憶體中存取的逐步運作。

物件導向(一) 物件與類別

發表於 2020-01-08 更新於 2022-10-11

要學習物件導向的第一步，就是要先釐清**物件(Object)與類別(Class)**的關係。
以下整理了幾個網路上比喻物件類別之間關係的例子:

  1. 類別是建築物的設計圖，而物件則是依照設計圖設計出來的建築物。
  2. 類別是一本武林秘笈，而物件則是精通武林秘笈的武林高手。
  3. 物件是實體化後的類別。

而我自己的理解是類別是虛擬的，需要將類別實體化(new出來)變成物件，才能被使用，下面將使用一段簡單的程式碼來示範何謂類別，何謂物件。

程式碼說明:

  1. 首先建立一個 Car 類別。
  2. Car 類別中包含三個屬性( brand、color、speed )及一個方法( Run )。
  3. 在主程式 Main 中實體化一個 Car 類別的物件名為 toyota。
  4. 賦予 toyota 這個物件的屬性值。
  5. 執行 Run() 方法，並將結果輸出。

以上就是物件與類別的基本介紹，可以將重點歸納為下面幾個項目:

  • 物件是實體化後的類別。
  • 類別中包含屬性( property )與方法( method )。
  • 要產生物件必須要將類別 new 出來。

將委派當作參數傳到其他方法

看到這裡您可能會想，那我就直接執行 MaMaCook() 就好啦，何須多此一舉宣告一個委派來執行。但呼應文章開頭所說的「委派就是將方法當成參數傳進另一個方法中」，在設計類別時，可能會碰到某個方法需要依照不同情境有不同的處理，但又不想把該處理用 if 或 switch 來處理，因為有可能變化很多，又或是無法預先得知處理的規則，這時我們就可以將方法指派給 howToCook，再將 howToCook 當作參數傳進其他 function。

例如下方範例的 GetFood 方法，其中可能有數十種，甚至數百種的烹煮食物邏輯，我們若把所有邏輯寫在同個 function 中，這個 GetFood 方法勢必會變得又臭又長，不好維護。所以我們將烹煮食物邏輯抽出，包裝在委派 HowToCook 中，並當成參數傳進 GetFood 方法，如此一來， GetFood 方法就只要專注於自己的輸出功能上，不用去在意烹煮食物邏輯。

在使用多重委派還有幾個特性需要注意:

  • 被指派的任何方法，一定都要符合 Delegate 宣告的 input output 規範。
  • 會依照被指派的順序依序執行。
  • 可以接受同樣的方法被重複指派。
  • 當執行委派參考時，得到的回傳值，會是最後一個被指派方法的回傳值。
